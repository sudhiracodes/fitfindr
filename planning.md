### Tool 1: search_listings

**What it does:**
Queries the listings database to extract secondhand garment items that closely match the user's natural language description keyword preferences, size requirements, and budget constraints.

**Input parameters:**
- `description` (str): Key matching phrases or style terms parsed from the user's request.
- `size` (str or None): The specific size filter (e.g., "M", "L").
- `max_price` (float or None): The maximum allowable price cap.

**What it returns:**
Returns a `list[dict]` containing matching item listings. Each dictionary record contains fields like `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

**What happens if it fails or returns nothing:**
If no inventory items match the filtered fields, it returns an empty list `[]`. The planning loop detects this empty response, stops further down-funnel tool operations immediately, sets a descriptive user error notification in the session, and aborts to the main interface.

---

### Tool 2: suggest_outfit

**What it does:**
Interfaces with the Groq LLM API to curate a unique and cohesive outfit styling layout combining the newly searched thrift item with pieces from the user's wardrobe.

**Input parameters:**
- `new_item` (dict): The target listing item selected from the search tool results.
- `wardrobe` (dict): The dictionary representation of the user's current closet structure containing structural listings under an `items` array key.

**What it returns:**
Returns a `str` outlining concrete styling configurations, specific mixing combinations, and quick accessorizing tips restricted under a 4-sentence ceiling limit.

**What happens if it fails or returns nothing:**
If the user's wardrobe profile data arrives empty or minimal, the system triggers an internal fallback branch. Instead of throwing an exception, it modifies the generation prompt instructing the LLM to yield universal, highly versatile styling advice and silhouette matching structures optimized for that specific item type.


---

### Tool 3: create_fit_card

**What it does:**
Formulates a trendy, creative, and engaging social media text caption layout based on the curated styling recommendation text and underlying thrift details.

**Input parameters:**
- `outfit` (str): The descriptive text layout generated by the `suggest_outfit` tool step.
- `new_item` (dict): The underlying thrift item dictionary record.

**What it returns:**
Returns a short `str` conversational caption formatted in aesthetic lower-case letters with 1–2 relevant emojis embedded.

**What happens if it fails or returns nothing:**
If the incoming outfit recommendation string is empty or incomplete, the tool guards against system crashes by returning a default aesthetic catch-all styling placeholder string.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent operates via an conditional state-evaluation flow rather than a rigid fixed sequence:
1. **Extraction**: The input natural language query is passed to an atomic parsing utility that handles regex checks to systematically isolate size tags and dollar amounts into structured search criteria fields.
2. **Evaluation**: The loop executes `search_listings`. Immediately following completion, the loop inspects the returned dataset. 
   - **Branch A (Fail)**: If the result list length equals 0, the loop flags an error in the session state object and breaks out early to return to the UI panel, completely skipping outfit compilation and caption generation.
   - **Branch B (Success)**: If valid items are found, the top result is assigned to the active session object, and execution advances smoothly to `suggest_outfit`.
3. **Completion**: Once the outfit recommendations are compiled, the state flows directly into `create_fit_card` to finalize the pipeline.
---

## State Management

**How does information from one tool get passed to the next?**
**How does information from one tool get passed to the next?**
Information is passed across a single centralized session dictionary object initialized at the beginning of the `run_agent()` entry execution. 

The tracked keys include:
* `query` (str)
* `parsed_criteria` (dict)
* `selected_item` (dict or None)
* `outfit_suggestion` (str or None)
* `fit_card` (str or None)
* `error` (str or None)

Tools write their results directly into dedicated keys inside this session dictionary wrapper. Subsequent tool functions extract their required parameters cleanly from these state-managed variables, ensuring data continuity without prompting the user for repeated input adjustments.

## A Complete Interaction (Step by Step)
FitFindr orchestrates a 3-step thrifting and styling pipeline. The process is triggered by a natural language query specifying a desired item, size, and budget. If `search_listings` locates matching items, the top result is stored in the session state and passed automatically to `suggest_outfit` alongside the user's wardrobe layout. Finally, the generated outfit composition is fed into `create_fit_card` to build an engaging, social-media-ready caption. If an upstream tool fails or yields empty results, the pipeline breaks early, populates an informative error context, and surfaces actionable guidance to the user.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The orchestration layer initializes a new session state dictionary. The query parser runs regex logic on the user input to extract constraints, setting `description="vintage graphic tee"`, `size=None`, and `max_price=30.0` inside `session["parsed_criteria"]`. 

The planning loop automatically calls the first tool: `search_listings(description="vintage graphic tee", size=None, max_price=30.0)`. The tool searches `listings.json` and finds a matching item. The loop confirms the result is not empty, selects the top match, and updates the state: `session["selected_item"] = {"id": 12, "title": "Faded Band Tee", "price": 22.0, "size": "M", "platform": "Depop", "brand": "Harley Davidson", "condition": "Good", "colors": ["Black", "Gray"], "description": "Perfect worn-in vintage look."}`.

**Step 2:**
The planning loop checks `session["selected_item"]`. Because it contains a valid item dictionary, the loop proceeds. It loads the active wardrobe context (`get_example_wardrobe()`) and calls the second tool: `suggest_outfit(new_item=session["selected_item"], wardrobe=wardrobe)`. 

The Groq SDK model (`llama-3.3-70b-versatile`) evaluates the item metadata against the user's closet and preferences. It generates an actionable recommendation, which the loop saves: `session["outfit_suggestion"] = "Pair this black Harley tee with your wide-leg denim and chunk sneakers for an effortless 90s grunge silhouette. Roll the sleeves up slightly and add a silver chain to tie the vintage textures together."`

**Step 3:**
The planning loop automatically routes the gathered context to the final creative layer by calling: `create_fit_card(outfit=session["outfit_suggestion"], new_item=session["selected_item"])` running on a high temperature setting (`0.95`). 

The model parses the breakdown text and builds a casual, lower-case social media text layout, updating the state: `session["fit_card"] = "thrifted this faded band tee off depop for $22 and honestly it was made for my wide-legs 🖤 full look in my stories"`.

**Final output to user:**
The Gradio web interface updates its three dedicated output panels using the finalized `session` dictionary properties:
* **Step 1 Panel (Top Search Match)**: Displays formatted text fields showcasing the $22 Faded Band Tee found on Depop.
* **Step 2 Panel (Outfit Suggestion)**: Displays the tailored 4-sentence styling guide combining the tee with wide-leg denim and chunk sneakers.
* **Step 3 Panel (Fit Card Caption)**: Displays the ready-to-copy, aesthetic social media caption: `"thrifted this faded band tee off depop for $22 and honestly it was made for my wide-legs 🖤 full look in my stories"`.
